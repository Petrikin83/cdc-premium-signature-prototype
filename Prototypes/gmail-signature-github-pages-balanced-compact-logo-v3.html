Self-review — Corporate Identity / Premium Email Design / Typography / Gmail Compatibility

  • ✓ Logo-to-identity spacing (v3 improvement): 18 px spacer between the logo and the identity text gives the brand mark more visual isolation. In v2 at 14 px the text felt glued to the badge. At 18 px both zones breathe independently — the logo reads as a self-contained mark, the identity reads as a separate text block. The 4 px reduction in identity column width (263 → 259 px) is imperceptible on the text.
  • ✓ Logo still properly anchors brand: 104 px retained from v2. The mark remains the visual primary on the left. The 21 px name is clearly a label within the brand zone, not a competing headline.
  • ✓ Floating divider (premium improvement): The separator is now a nested 1 × 64 px table cell in #eeeeee, centered via valign="middle" on the outer 1 px column. The row height is driven by the logo cell (~104 px image + 28 px padding = ~132 px total). The 64 px colored line sits centered in that row with ~34 px white breathing room above and below. The divider does not reach the horizontal rule below — the horizontal rule is in a separate row entirely. #eeeeee is lighter than the horizontal rules (#e0e0e0), reinforcing that it is a soft zone separator rather than a structural border.
  • ✓ Divider does not touch horizontal rules: The floating divider is inside ROW 1 only. ROW 2 is the horizontal divider, a completely separate table row. No visual connection between the two.
  • ✓ Award banner secondary: 184 px retained from v2. Reads as credential, not hero block.
  • ✓ Typography harmonious: 21 / 13 / 13 px stack unchanged. Contact 13 px consistent with secondary text scale. Disclaimer 10 px / 1.45 / #666666 clearly tertiary, left-aligned.
  • ✓ Copy safety: The preview-chrome div's padding is CSS-only — it creates visual spacing in the browser but does not produce copyable whitespace HTML around the signature table. The signature table starts immediately as the first child element inside preview-chrome. Selecting from inside the white card boundary and pasting into Gmail will capture only the table. The copy-note above the card instructs the user to select from inside the card, not from outside it.
  • ✓ Gmail compatibility: All CSS inline. No style blocks, web fonts, base64, SVG. table-layout: fixed, explicit width/bgcolor on all cells. Nested table for divider is standard email-safe technique. colspan="6" correct across all rows.
